# Alma Carita — лична платформа на Карина Вълканова

Не просто сайт за имоти, а **личната платформа на Карина** — психолог, педагог и брокер. Четири стълба, събрани около нея, представени като венчелистчетата на едно цвете (логото).

Това е **Фаза 1**: пълна публична част с immersive дизайн + admin панел, върху mock данни. Архитектурата е модулна — преходът към реална база (Supabase) във Фаза 2 не пипа UI.

---

## Бърз старт

Node.js 18.17+ (препоръчително 20+).

```bash
npm install
npm run dev      # http://localhost:3000
```

Production:
```bash
npm run build && npm start
```

## Концепцията — цветето с 4 листа

Логото на Alma Carita е четирилистно цвете с чакра-цветове. Всяко венчелистче е стълб; центърът е Карина.

```
            🟢 Имоти
               │
   🟡 Събития ─ ❤ ─ 🔵 Обучения
               │       (Карина = център)
         🟠 Детски център
```

| Венчелистче | Цвят | Стълб | Маршрут |
|---|---|---|---|
| 🟢 горно | `#6aaa7a` | Имоти | `/imoti` |
| 🔵 дясно | `#7a8fbb` | Обучения | `/obuchenia` |
| 🟠 долно | `#e07848` | Детски център | `/detski-tsentar` |
| 🟡 ляво | `#d4b840` | Събития | `/sabitia` |
| ❤ център | `#c04060` | Карина | `/za-men` |

Началната страница е **scrollytelling**: започва с историята на Карина, после анимираното цвете-навигатор, после вратите към стълбовете.

## Immersive дизайн

Цветове и шрифтове остават от одобрения бранд (Cormorant Garamond + Jost, cream/lavender/blush/sage/amber + чакра). Новото е **движението**:

- Scrollytelling с reveal-on-scroll (`lib/utils/useScrollReveal.ts`)
- Анимирано цвете-навигатор с 3D parallax tilt (`components/public/FlowerNav.tsx`)
- Морфиращи blob фонове, petal-open анимации
- Per-pillar акцентни цветове (`.pillar-*` класове)
- `prefers-reduced-motion` се уважава навсякъде

Дълбочината е постигната с CSS 3D transforms и scroll-linked ефекти — **без** Three.js/WebXR, защото това дава желаното усещане без излишна тежест.

## Архитектура — защо е scalable

UI никога не чете данни директно — само през `lib/data/queries.ts` (репозиторий). Мутациите минават през server actions в `lib/actions.ts`.

```
UI → lib/data/queries.ts → (сега) mock.ts / trainings.ts / events.ts
                          → (Фаза 2) Supabase — само телата се сменят
```

### Структура

```
app/(public)/      11 публични страници
app/(admin)/       admin панел
lib/
├── types.ts       Property, Inquiry, Training, EventItem
├── data/
│   ├── queries.ts ← РЕПОЗИТОРИЙ
│   ├── mock.ts    8 имота, запитвания
│   ├── trainings.ts  обучения
│   └── events.ts  минали събития
├── actions.ts     server actions (вкл. интерес към обучение/център)
└── utils/         формати, любими, scroll-reveal
components/public/  + FlowerNav, HomeExperience, Trainings*, Events*, ChildrenCenter
components/admin/   таблици, форми, inbox, QR
```

## Ограничения на прототипа

- **Данните не персистират** — mock в памет; смяна на статус/изтриване работят за сесията, връщат се при рестарт. Трайността идва със Supabase.
- **Снимките са градиенти**, не реални файлове.
- **Логинът е симулиран.**
- Обученията/събитията са примерни, базирани на реалния опит на Карина — конкретните детайли (дати, цени, локация) тя допълва.

## Фаза 2 — пътна карта

1. `npm install @supabase/supabase-js @supabase/ssr`
2. Разкоментирай `lib/supabase/client.ts`
3. Таблици: `properties`, `property_images`, `inquiries`, `trainings`, `events`, `clients`
4. Замени mock четенето в `queries.ts` (signatures остават)
5. Supabase Auth към логина + middleware за `/admin`
6. Качване на снимки (Storage) вместо градиенти
7. Admin CRUD за обучения и събития

Виж **`DEPLOY.md`** за Vercel + Supabase инструкции.
